webpack中的publicPath

Posted on 2019-12-05 | In 前端技术 | Visitors:
Words : 1.3k | Reading ≈ 5 min

webpack的常用基本配置我们可能已经耳熟能详，比如 input,output,module,plugins,devServer的配置等等。而在这些基本配置中，其实还有一些细节参数，它可以帮助我们更好的定制化自己的打包目录结构，但它可能并不是那么好理解，比如publicPath。

webpack官网这么解释：
publicPath配置项在很多场景下都非常有用，它允许你给你的应用中的所有静态资源指定一个基本路径。

The publicPath configuration option can be quite useful in a variety of scenarios. It allows you to specify the base path for all the assets within your application.

有些抽象，我总结了下，它大概做的事情就是：可以帮我们把不同文件生成到不同的目录下，并处理好资源引用的url路径。

什么时候需要使用publicPath?

1. 希望对打包出来的文件进行目录结构划分

webpack打包出来的文件，默认都统一放在output配置的path路径下，项目稍大一点，这个目录中的文件就比较杂乱了，我们可能会希望给这些文件进行归类。当然我们可以粗暴一点通过filename来指定一个子目录。但是，在这这个子目录中，文件还是混在一起没有层级，这个时候我们需要配置相应plugin或者loader的publicPath来自定义产出文件的目录结构。

需要注意的是，devServer中还有一个叫做contentBase的参数，这个参数如果配置的不好，跟publicPath一搭配，很可能会导致请求不到页面（我也是被这个坑了很久）。这个地方如果出问题，基本上原因在于contentBase设置的路径范围太小了，去掉contentBase配置，或者给它配置多个路径，把输出目录包含进来，就可以解决问题。
